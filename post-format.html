Post Format

The format of emails, let's call them messages, should probably be XML. I know XML Sucks, but the other choices don't fit well. A message, in my opinion, is fundamentally a document, and any formatting or links in that document would be better suited as markup than anything. I think the problem of namespace musical chairs can be resolved if you have a way to look them up by content hash. Documents can be made immutable, links and all in fact, if they link by content hash key. Avoiding some of the XML pitfalls such as

  • use base64 embedded data fucking anywhere
  • require javascript/non-XML
  • use the contents of tags as arguments, not as non-regular content
  • use multiple namespaces (you can make a namespace that just combines the elements from the ones you like into a single namespace as a static resource)
  • have stuff in it that indicates programmatic behavior
  • depend on whitespace
  • embed XML inside XML (this counts as multiple namespaces)
  • use XPath
  • have different tags for linking to different types of files.
  • have different tags to indicate style instead of CSS
  • have elements with different functions use the same tag.
...then I should be just fine. Please convince me differently! ;_;

I'm also more familiar with XML, grudgingly, then say YAML or BEncoding, and there is more documentation about how to use XML than just about anything else.

One thing I will not do is use RFC822. If you're going to have to assume a message is RFC822 content type, to tell you that it's the same content-type as every other message, you might as well just assume that message is in that content type. A message's links can have other content types, but the message itself should be self descriptive, acting as the base for all content type determination.

Sound good? Please anything I should change or do, or not do, let me know.
